# esdeath 构建系统功能实现总结

本文档记录了从 surge-master-2 迁移到 esdeath 的所有核心功能实现。

## 核心原则

- **完整功能对等**：100% 复刻 surge-master-2 的功能，不增加任何额外特性
- **仅支持 Surge 格式**：不实现其他平台的写入策略
- **保持原有优化**：域名去重、IP CIDR 归并、关键词过滤等

## 已实现功能清单

### 1. 基础架构

- ✅ FileOutput 基类：支持所有 Surge 规则类型，内置优化逻辑
- ✅ WritingStrategy 接口：仅实现 SurgeWritingStrategy
- ✅ 规则类型支持：DOMAIN、DOMAIN-SUFFIX、DOMAIN-KEYWORD、URL-REGEX、USER-AGENT、IP-CIDR、IP-CIDR6、GEOIP、IP-ASN、PROCESS-NAME、AND、OR、NOT、SUBNET、DEST-PORT、SRC-IP、IN-PORT、PROTOCOL、RULE-SET、DOMAIN-SET、SCRIPT
- ✅ Trace 系统：性能追踪和分析

### 2. TLD 验证系统（无 tldts）

- ✅ 三层 TLD 验证机制：
  1. ICP_TLD 白名单（131 个中国域名后缀）
  2. domestic.conf 自动白名单提取
  3. global.conf 验证处理
- ✅ 验证结果：151 ICP + 18 domestic = 169 个白名单 TLD

### 3. 域名活性检测

- ✅ 使用 foxts/retrie 进行 WHOIS 关键词过滤
- ✅ 使用 foxts/pick-random 随机选择 DoH 服务器
- ✅ 使用 @henrygd/queue 实现 32 并发队列
- ✅ 使用 cli-progress 显示检测进度
- ✅ 多层次验证策略：
  - Apex Domain 检查
  - DNS 查询（A/AAAA/CNAME）
  - DoH 查询（国际+国内服务器）
  - WHOIS 验证（NS/MX/TXT + 关键词过滤）
- ✅ 性能优化：结果缓存、互斥锁、进度显示
- ✅ 验证脚本只输出 JSON，不自动清理

### 4. IP 规则优化

- ✅ IPListOutput 类：专门处理 IP 相关规则
- ✅ 自动 CIDR 归并（fast-cidr-tools）
- ✅ 支持 IPv4 和 IPv6 分别优化
- ✅ 支持排除指定 CIDR
- ✅ cli-progress 进度显示

### 5. 规则优化机制

- ✅ 域名去重：HostnameSmolTrie 支持通配符和子域名
- ✅ IP CIDR 归并：fast-cidr-tools 自动合并
- ✅ 端口去重：使用 Set 数据结构
- ✅ ASN/GeoIP 去重：使用 Set 数据结构
- ✅ 关键词过滤：Aho-Corasick 算法（foxts/retrie）

### 6. 性能优化

- ✅ 增量构建：基于文件内容哈希的缓存系统
- ✅ 并行处理：Promise.all 批量任务执行
- ✅ 流式处理：fetchTextByLine 支持大文件
- ✅ 缓存系统：.cache 目录存储元数据
- ✅ 进度显示：cli-progress 统一进度条

### 7. 构建脚本

- ✅ build-reject-domainset.ts：从 AdGuard 过滤器构建拒绝域名集
- ✅ build-common.ts：从 Source 目录构建通用规则集
- ✅ validate-global-tld.ts：TLD 验证脚本
- ✅ validate-domain-alive.ts：域名活性检测脚本
- ✅ download-previous-build.ts：从 GitHub Release 下载上次构建

### 8. 主构建流程

- ✅ 并行任务执行
- ✅ 构建时间追踪
- ✅ 输出目录：public/List 和 public/Domainset
- ✅ 仅生成 Surge 格式文件

## 技术栈对比

| 特性       | surge-master-2        | esdeath          |
| ---------- | --------------------- | ---------------- |
| TLD 验证   | @adguard/tldts        | 自实现三层验证   |
| 域名去重   | mnemonist/trie        | HostnameSmolTrie |
| IP 归并    | fast-cidr-tools       | fast-cidr-tools  |
| 关键词过滤 | @napi-rs/aho-corasick | foxts/retrie     |
| 并发控制   | @henrygd/queue        | @henrygd/queue   |
| 进度显示   | cli-progress          | cli-progress     |
| 缓存系统   | 基于 mtime            | 基于内容哈希     |

## 移除的功能

- ❌ 流媒体服务构建（Netflix、Disney+ 等）
- ❌ 其他平台支持（Clash、sing-box 等）
- ❌ 自动清理死域名（只检测不清理）

## npm 脚本

```bash
# 构建相关
npm run build           # 主构建流程
npm run build:reject    # 构建拒绝域名集
npm run build:common    # 构建通用规则集

# 验证相关
npm run validate:domain-alive  # 域名活性检测
npm run test:tld              # TLD 验证测试

# 其他
npm run sync           # 同步规则和模块
```

## 注意事项

1. 域名验证只检测不清理，需要人工处理
2. 增量构建基于内容哈希，首次运行会构建所有文件
3. 所有进度条使用 cli-progress 统一风格
4. 仅支持 Surge 格式输出

## 核心实现文件

- `/lib/rules/base.ts` - FileOutput 基类
- `/lib/rules/ip.ts` - IPListOutput 类
- `/lib/is-domain-alive.ts` - 域名活性检测
- `/lib/create-file.ts` - 增量构建系统
- `/scripts/validate-global-tld.ts` - TLD 验证
- `/scripts/validate-domain-alive.ts` - 域名验证脚本
